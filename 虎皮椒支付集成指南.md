## 配置说明

### 测试环境配置（默认）

无需修改，使用虎皮椒提供的测试配置：

```env
NODE_ENV=development
XUNHUPAY_TEST_PID=2000000000
XUNHUPAY_TEST_KEY=test_key_1234567890
```

测试环境特点：
- 不会产生真实扣款
- 可以随意测试
- 使用虚拟订单号

### 生产环境配置

#### 1. 注册虎皮椒账号

1. 访问 [虎皮椒官网](https://www.xunhupay.com/)
2. 点击"立即注册"
3. 填写注册信息并完成实名认证

#### 2. 获取商户配置

登录后进入"商户中心" → "API 设置"，记录以下信息：

- **商户 ID (pid)**: 类似 `2000000000`
- **API 密钥 (key)**: 32位字符串

#### 3. 配置环境变量

编辑 `payment-backend/.env` 文件：

```env
# 生产环境
NODE_ENV=production

# 替换为你的真实配置
XUNHUPAY_PID=your_real_pid_here
XUNHUPAY_KEY=your_real_key_here

# 回调地址（必须使用 HTTPS）
NOTIFY_URL=https://yourdomain.com/api/payment/notify
RETURN_URL=https://yourdomain.com/subscribe?success=true

# 服务器端口
PORT=3001
```

#### 4. 配置回调地址

在虎皮椒"商户中心" → "API 设置"中，设置：

- **异步通知地址**: `https://yourdomain.com/api/payment/notify`
- **同步跳转地址**: `https://yourdomain.com/subscribe?success=true`

⚠️ **重要**：生产环境必须使用 HTTPS，否则支付回调可能被拦截。

## 生产环境部署

### 1. 部署前端

```bash
# 构建生产版本
npm run build

# 部署 dist 目录到你的服务器
# 可以使用 Nginx、Apache 或任何静态文件服务器
```

### 2. 部署后端

```bash
# 安装生产依赖
cd payment-backend
npm install --production

# 使用 PM2 启动
npm install -g pm2
pm2 start server.js --name "writing-master-payment"

# 或使用 forever
npm install -g forever
forever start server.js
```

### 3. 配置 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name yourdomain.com;

    # 前端静态文件
    location / {
        root /path/to/writing-master/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://localhost:3001/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 4. 配置 HTTPS（必需）

使用 Let's Encrypt 免费证书：

```bash
# 安装 Certbot
sudo apt-get install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d yourdomain.com

# 自动续期
sudo certbot renew --dry-run
```

### 5. 更新前端 API 配置

修改 `src/utils/api.ts`：

```typescript
// 开发环境
const API_BASE_URL = 'http://localhost:3001';

// 生产环境（取消注释）
// const API_BASE_URL = 'https://yourdomain.com/api';
```

## 常见问题

### Q1: 支付服务不可用怎么办？

**A:**
1. 检查后端服务是否启动：访问 http://localhost:3001/api/payment/test
2. 检查端口是否被占用：运行 `netstat -ano | findstr :3001`
3. 查看后端日志，检查是否有错误

### Q2: 支付成功但会员未开通？

**A:**
1. 检查异步通知地址是否正确配置（必须使用 HTTPS）
2. 检查 API 密钥是否正确
3. 查看后端日志，检查是否收到回调
4. 清除浏览器缓存并重新登录

### Q3: 测试环境和生产环境的区别？

**A:**
- 测试环境使用虚拟账号，不产生真实扣款
- 生产环境使用真实账号，会产生真实交易
- 两者的 API 密钥和商户 ID 不同

### Q4: 如何获取真实的虎皮椒商户 ID？

**A:**
1. 访问 https://www.xunhupay.com/
2. 注册账号并完成实名认证
3. 进入"商户中心" → "API 设置"
4. 查看商户 ID 和 API 密钥

### Q5: 手续费是多少？

**A:**
- 虎皮椒手续费：0.3%-0.5%（按交易金额）
- 微信支付/支付宝由虎皮椒代收，无需单独申请

### Q6: 支持退款吗？

**A:**
虎皮椒支持退款，需要：
1. 联系虎皮椒客服
2. 提供订单号和退款原因
3. 虎皮椒会在 1-3 个工作日处理

### Q7: 如何查看支付记录？

**A:**
1. 登录虎皮椒商户后台
2. 进入"交易记录"
3. 可以查看所有订单和支付状态

### Q8: 支付回调地址必须用 HTTPS 吗？

**A:**
是的，生产环境必须使用 HTTPS，否则：
- 支付宝会拒绝回调
- 微信支付可能会拦截回调

### Q9: 可以同时支持微信和支付宝吗？

**A:**
可以！虎皮椒同时支持：
- 微信支付（公众号支付、H5支付、扫码支付）
- 支付宝（电脑支付、手机支付）

用户可以在支付页面选择任意一种方式。

### Q10: 如何调试支付回调？

**A:**
1. 在生产环境，查看后端日志
2. 使用虎皮椒商户后台的"回调记录"
3. 使用测试环境模拟支付（不会产生真实扣款）
